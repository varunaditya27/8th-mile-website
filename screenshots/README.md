## Screenshot Guidelines

- **Resolution**: 1920x1080 for desktop, 375x812 for mobile
- **Format**: PNG with transparency where applicable
- **Quality**: High-resolution, clear and crisp
- **Content**: Show real or realistic demo data
- **UI State**: Show the interface in an active, engaged state

## File Naming Convention

- Use lowercase with hyphens for spaces
- Include device type for mobile screenshots
- Example: `event-details-mobile.png`, `homepage-desktop.png`

## Adding Screenshots

1. Take screenshots of the application in use
2. Edit and optimize the images if needed
3. Save them in this directory with appropriate names
4. Update the README.md file references if necessary

## Tools for Screenshots

- **Desktop**: Built-in screenshot tools, Lightshot, or similar
- **Mobile**: Device screenshots or browser dev tools
- **Editing**: GIMP, Photoshop, or online tools for optimization
